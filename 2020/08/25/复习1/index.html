–数据库：mysql 5.7，oracle 11g

–连接工具：Navicat，PLsql

–DML：select update delete insert ；测试工程师经常用到，一定要拿实例
–语法：去重（distinct） 模糊查询（like） 多少之间（between） 包含（in） 不包含（not in）
–查询：如何查询今天一天的订单数量（to_date:讲字符串转换为时间类型）
select count() from ecs_order where to_char(order_time,’yyyy-mm-dd’)=’2019-08-07’
–如何查询这个月的订单数量（to_char:讲时间类型转换为字符类型）
oracle:
select count() from ecs_order where to_char(order_time,’yyyy-mm’)=’2019-04’;

mysql:
select count(*) from ecs_order where order_time between ‘2019-01-01 00:00:00’ and ‘2019-01-01 11:59:59’
备注：mysql不能用to_date和to_char

–如何查询订单表内相同商品的数据

–练习1：查询出至少有两名秘书的所有部门的部门号，并按人数降序排序。
select deptno from emp where job =’CLERK’ group by deptno having count()>=2 order by count() desc;
select deptno from emp group by job,deptno having job=’CLERK’ and count()>=2 order by count() desc;
–sql执行顺序：from-where-group by-having-select-order by
–多表查询
1.分析题目涉及哪几张表；
2.如果查询的数据来自于多张表，找到这些表的相同字段，在where后面，把这些字段用等于号连接起来；
3.如果需要对单条记录进行过滤，就把过滤条件追加到where子句后面，用and连接起来；
–多表查询(内连接，外连接，全连接)
–内连接：inner join （等值查询）,连接条件的字段，双方都有相同的数据才展示
select e. , d. from emp e ,dept d where e.deptno = d.deptno;
select e. , d. from emp e inner join dept d on e.deptno = d.deptno;
–外连接(左连接left join，右连接right join)
–左连接：连接条件的字段，左表数据都要显示，以及右边和左边相同的数据才展示
select e. , d. from emp e ,dept d where d.deptno = e.deptno(+);
select e. , d. from dept d left join emp e on e.deptno = d.deptno;
–右连接：连接条件的字段，右表数据都要显示，以及左表和右表相同的数据才展示
select e. , d. from emp e ,dept d where e.deptno(+) = d.deptno;
select e. , d. from emp e right join dept d on e.deptno = d.deptno;
–全连接(full join）连接条件的字段，双方只要有的数据都显示
–oracle写法：
–rownum：伪行，只有在查询时候才出现
select e.,rownum from emp e ;
–查询emp前五行的数据
select from emp where rownum <= 5;
–查询emp6-10行的数据(分页查询)
select from (select e.,rownum t from emp e where rownum <=10) x where x.t>=6
–查询最后5行的数据(最新5行)
select from (select from emp order by rownum desc) where rownum <=5;

–mysql写法：
–查询emp前五行的数据
select from emp limit 0,5;
–查询emp6-10行的数据(分页查询)
select from emp limit 5,5;
–查询最后5行的数据(最新5行)
select * from zt_bug order by id desc limit 0,5;

–三种删除的区别
delete:删除数据，可以接where指定删除一行记录，会发起事务
truncate:删除整张表的数据，表结构还在，不会发起事务
drop：删除整张表
–新增数据

–修改数据
update 表名 set 列名=值,列名=值 where 过滤条件
–删除数据
delete 表名 where 过滤条件
–提交事务
commit;
–回滚事务
rollback;

–你搭建测试环境有遇到什么问题吗？
（tomcat）
答：我搭建完成之后，无法启动tomcat服务，发现是tomcat端口号被占用，tomcat默认8080，进入tomcat的conf文件夹下，修改文件server.xml，修改默认端口号，保存并退出

(Apache)
答：我搭建完成之后，在网页输入项目的地址，然后报错显示http状态码403，服务器拒绝访问，因为项目某些文件没有权限，然后定位发现，/var/www/html文件下项目文件夹，当时只用了chmod 777 项目文件名，只给文件夹加了所有权限，没有给文件夹内所有文件加权限，解决：chmod 777 -R 项目文件名

–测试环境有过什么问题吗？
答:磁盘满了，导致服务器访问卡顿或者无法访问，解决：登录linux系统查看磁盘信息，然后清理磁盘（一般回答前要说明，清理前和上级已经说明这个情况，要不然面试官会认为你没有安全意识，随意清理服务器的文件）

–理论
svn：代码管理工具
Git：代码管理工具
osi七层模型：物理层，数据链路层，网络层，传输层，会话层，表示层，应用层
tcp:建立连接再传输数据，传输性可靠，效率低；建立连接需要3次，断开连接4次 （打电话）
udp:不需要建立连接，传输数据带目的地址，不可靠，效率高 （发短信）
hosts文件:C:\Windows\System32\drivers\etc\hosts
web基础架构：页面布局，代码，数据库，服务平台，BS架构
服务器操作系统：linux
web服务器：Apeche，tomcat，Nginx
关系型数据库：mysql，oracle，db2
前端代码：html，css，JavaScript
后端代码：java，php
服务平台：阿里云，腾讯云
HTTP协议：在应用层，超文本传输协议，由请求和响应构成的，http是无状态
HTTP常见的请求方式有哪些：get和post，put，delete
get和post区别：get的请求参数在url里，一般用于获取数据，没有请求body，有请求头
post请求参数在请求body，一般用于提交数据，什么都有
HTTP响应状态码：在响应行内
1：服务器收到请求，需求请求者执行操作
2：请求成功并且处理
3：重定向，需要进一步操作完成
4：客户端错误
5：服务器错误
常见的状态码有哪些：
200：请求成功
301: 永久移动
302：临时移动
404：无法根据客户端的请求找到资源
500：服务器内部错误
503：服务器超载或者维护

–测试理论
软件测试定义：在规定的条件下，对程序进行操作，以发现程序的缺陷，并且对程序评估，提高软件的质量

软件测试目的：软件测试不仅仅是为了发现软件缺陷与错误，而且也是对软件质量进行度量和评估，以提高软件的质量

软件测试的具备条件：良好的表达沟通能力，编辑文档能力，思维逻辑能力，耐心，细心，抗压能力，有责任心，技术不错，独立解决问题，具有怀疑精神

你的优势在哪里：耐心，细心，有责任心，以及能够独立完成一个项目的测试，有过带实习生的经验，能够独立，主动解决问题，学习能力非常好；

–测试类型

按技术划分：白盒测试，黑盒测试
其他测试：回归测试，冒烟测试，探索性测试，灰度测试
单元测试 SIT（集成测试） UAT(验收测试)

–质量六大特性
1.功能性 2.可靠性 3.易用性 4.效率 5.可维护性 6.可移植性
重点：电梯测试点，桌子测试点，杯子测试点，笔，鼠标，花盆
下单整个流程的测试点（注册，登录，购物车，结算，下单，支付，上传，搜索）
–需求评审
对需求进行查漏补缺，如何实现这个需求，有疑问的，错误的，模糊，遗漏给产品提出相关意见，然后修改
举例：你在需求评审上提过什么意见？
比如：我们当时做了一个优惠卷功能，产品讲完后，我发现没有说使用优惠券再取消订单，是否返回优惠卷

–测试计划
自己独立写过测试计划
测试计划的内容：项目背景，项目测试范围，测试方法/策略,测试资源（测试工具），开始和结束条件，进度安排，组织架构
测试风险（人员风险，进度风险，质量风险，需求变更）

–测试需求分析
使用x-mind思维导图将每个功能点进行细分，提起测试点，避免遗漏

–编写测试用例
测试用例要素：编号，名称，优先级，前置条件，操作步骤，预期结果….

测试用例采用的方法：等价类，边界值，错误推测法，判定表，场景法（举实例） 我们没讲的（因果图，正交表）

如何保持测试用例覆盖率：1.我会对需求进行充分的理解和熟悉，然后用x-mind对需求进行分析细化 ，构造思维导图提取测试点2.我写完用例会自己检查一次 3.我们会进行用例评审会议

如何写好测试用例：1.覆盖所有需求 2.考虑正常异常场景 3.颗粒度要均匀 4.操作步骤要足够详细，要素齐全 5.用例评审，及时更新测试用例

–用例评审
对着用例，把一条条的用例读出来，参加人员:开发，产品，测试
但是某些公司，是直接发给测试老大看下
还有是发给产品看下，让他们找出问题

–准备数据
我们写完测试用例后，并且评审，开发还没有提交新的测试版本，我会准备一些测试数据

–执行测试
1.执行冒烟测试，如果不通过，打回让开发重新修复，自测完再给我们
2.开发认为不是bug，你认为是bug，如何处理？

4.如何看待开发和测试的关系？

6.开发压缩了测试的时间，怎么处理？
7.开发做出来的产品和需求不一致，怎么处理？
8.如果有一个严重bug没处理，是否能上线？
9.如果遇到解决不了bug，如何处理？
10.测试时间不够，如何完成测试？
11.难以复现的bug如何处理？

–测试报告
包含内容：数据统计（bug，用例，人力），遗留bug情况，测试风险（测试中发现的问题）
测试对象评估（功能，ui，用户体验，兼容），测试结论，测试总结

–测试流程
需求评审-测试计划-需求分析-测试用例-用例评审-搭建环境-执行测试-回归测试-测试报告-验收-上线
