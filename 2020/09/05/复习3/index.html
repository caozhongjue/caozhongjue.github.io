#面向对象有三大特性

#封装：定义一个类和方法，不用管内部结构，直接用就行了

#继承：重新定义一个类，然后它属于什么父类下，可以直接用父类的属性和方法

#多态：继承了父类之后，重写父类的方法

#讲一下类，方法，对象的理解

#类：类似一张图纸，有一定属性和功能（方法）

#方法：类里面的函数

#对象：根据这个图纸，实际做出来的一个实体
class man(object):
hand=’两只’

def game(self):
    print('在地上玩')

class superman(man):

def game(self):
    print('在天上玩')

#创建一个类对象ZYW,实例化对象
ZYW = man()

#调用类的属性，并打印
print(ZYW.hand)

#调用类的方法
ZYW.game()

#创建一个类对象DW,实例化对象，是man的子类
DW = superman()

#调用man类的属性，并打印
print(DW.hand)

#调用man类重写的方法
DW.game()

#类里面的初始化函数（构建函数）：def init
class man(object):

#__init__构建函数
def __init__(self,name,age):
    self.name=name
    self.age=age

def say(self):
    print('我的名字叫%s，我的年龄是%d' %(self.name,self.age))

DW = man(‘杜伟’,18)
DW.say()

#类里面的局部变量变为全局变量
class x(object):
def abc(self):
self.a = 18

    print('我的年龄是',self.a)

DW.ccc()

–web端UI自动化
python+selenium+unittest+jenkins
python：一种面向对象的编程语言
selenium：UI的自动化工具，也可以叫做第三方库
unittest：是一种测试框架，是python自带的内置模块
jenkins:是一个持续集成的工具，是定时任务工具

1、值是动态值
2、元素未加载
3、停顿时间过长
4、值在另一个表单
5、值在另一个窗口
6、隐藏元素
7、元素不唯一
8、界面有变动

1、语法不一样；
2、CSS定位比较稳定。
3、一般情况下定位速度css要比XPath快

–unittest这个框架怎么样理解的
unittest内置模块

###导入模块###
import unittest

###新建一个类，继承unittest里的TestCase类###
class mytest(unittext.TestCase):

###然后重写这个类里面的方法
def SetUp(self):初始化环境，一个用例文件有多少条用例，它方法下的内容就要执行多少次
def test(self)：测试用例，必须以test开头，后面可以随遍写，执行顺序1-9，a-z，可以写多条用例
断言：必须相同(assertEqual(a,b)),包含(assertIn(a,b))
def tearDown(self)：恢复环境，清理一些不用的数据，有多少用例，它方法下的内容就要执行多少次，一般用来退出浏览器
类里面一个方法下的变量需要全局使用，需要再前面加self

###执行用例的方法###
unittest.main()
执行用例的顺序：SetUp-test01-tearDown，SetUp-test02-tearDown
–测试套（测试工程）
三个文件夹，一个文件
public文件夹：存放公共的用例，封装起来，经常用到的用例，更好的维护用例脚本
testCases文件夹：存放测试用例，里面的py文件用的是unittset框架编写
reports文件夹：存放我们执行完用例后的测试报告
runAllCases文件：它的作用是执行testCases文件夹内所有的测试py文件，并将结果放在reports内
结果以html的形式展示，用的第三方工具HTMLTestRunner.py文件（备注：这个文件要放在python安装路径lib目录下）
最后用jenkins完成定时任务
